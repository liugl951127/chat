# 金融合规多端聊天与交易系统 — 总体方案

> **技术栈**：Java 17 + Spring Boot 3 + Spring Cloud Gateway + Vue 3 + 国密硬件加密机 + WebSocket
> **合规基线**：等保 2.0 三级 + 《个人信息保护法》 + 《证券期货投资者适当性管理办法》 + 银保监/证监会销售留痕
> **核心复用**：OpenClaw Enterprise (17 模块企业级底座) + 加固 6 个新模块

## 2. 六大新模块速览

| 模块 | 端口 | 核心职责 |
|------|------|----------|
| **auth-center** | 8081 | 微信/小程序/企微/H5 登录 + OpenID 统一 + JWT 颁发 + 实名绑定 |
| **chat-center** | 8082 | WebSocket 长连接 + 消息持久化 + 历史回溯 + 联网核查引擎 |
| **trade-gateway** | 8083 | 交易指令风控拦截 + 短信二次验证 + 国密签名 + 交易流水 |
| **compliance-center** | 8090 | 适当性匹配 + 风险测评 + 留痕规则 + 监管报送 |
| **kms-gateway** | 8091 | 国密加密机代理 (SM2/SM3/SM4) + 密钥生命周期 |
| **audit-center** | 8093 | 全链路审计 + 哈希链存证 + 监管查询接口 |

## 4. 数据流与留存策略

| 数据类型 | 存储 | 留存期 | 加密方式 |
|----------|------|--------|----------|
| 会话消息 | MongoDB (分片) | 5 年 | 字段级 SM4 |
| 客户身份 (手机/身份证) | MySQL | 永久 | SM4 + 国密机托管密钥 |
| 交易流水 | MySQL 交易库 | 永久 | 哈希链 (Merkle) |
| 审计日志 | MySQL + 冷归档 | 10 年 | 不可改 + 时间戳 (TSA) |
| 联网核查结果 | MySQL + ES | 3 年 | 字段级 SM4 |
| 短信验证码 | Redis (60s TTL) | 60 秒 | SM3 哈希 |
| 登录凭证 | Redis (refresh token) | 14 天 | 哈希存 |

## 5. 合规清单 (28 项)

### 5.1 等保 2.0 三级
1. 双向 TLS (mTLS)
2. 4A 统一 (认证/授权/审计/账号)
3. 重要操作双人复核
4. 网络区域划分 (DMZ/内网/工控)
5. 漏洞扫描 + 渗透测试
6. 数据备份 (3-2-1 策略)
7. 日志保留 ≥ 6 个月

### 5.2 个人信息保护法
8. 隐私政策弹窗 + 同意记录
9. 敏感个人信息 (身份证/手机) 单独同意
10. 数据最小化采集
11. 客户查询/更正/删除权
12. 数据出境评估 (跨境业务)
13. 个人信息影响评估 (PIA)

### 5.3 金融销售留痕
14. 销售全过程录音录像 (双录)
15. 适当性匹配留痕
16. 风险测评有效期管理 (12 个月)
17. 客户身份实名 + 弱实名分层
18. 交易关键环节二次验证
19. 营销宣传材料审核留痕
20. 投资冷静期 (24h) 应用
21. 异常交易实时监测

### 5.4 密码合规 (GM/T 系列)
22. 国密 SM2/SM3/SM4 全覆盖
23. 密钥全生命周期 (生成/分发/存储/销毁) 在加密机
24. 密钥与系统分离
25. FIPS 140-2 Level 3+ 硬件

### 5.5 业务连续性
26. 同城双活 + 异地灾备 (RPO 5min / RTO 30min)
27. 演练记录 (季度)
28. 监管报送通道 (T+1 自动化)
